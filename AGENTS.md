# GAIA 개발 가이드

이 저장소는 Next.js와 Supabase를 기반으로 하는 웹 게임 `gaia` 프로젝트입니다. 사용자는 이름을 입력해 게임을 시작하고, 회전하는 지구 위에 발생하는 자연 파괴 이벤트를 제한 시간 안에 올바른 스킬로 해결합니다.

## 기본 원칙

- Next.js App Router를 기본 구조로 사용합니다.
- TypeScript를 사용하고, 새 코드에는 명시적인 타입을 둡니다.
- 서버 상태와 영속 데이터는 Supabase를 기준으로 설계합니다.
- 게임의 핵심 상태는 클라이언트에서 즉시 반응하도록 관리하되, 랭킹과 업적 기록은 서버를 통해 저장합니다.
- 문서는 `docs/` 아래에 유지하고, 기획 변경이 생기면 관련 문서를 같이 갱신합니다.

## 예상 기술 스택

- Framework: Next.js
- Language: TypeScript
- UI: React, CSS Modules 또는 Tailwind CSS 중 프로젝트 초기 설정에서 하나 선택
- 3D 지구: Three.js 또는 React Three Fiber
- Database/Auth/API: Supabase
- Validation: Zod
- Test: Vitest, Playwright

## 구현 방향

- 첫 화면은 이름 입력과 시작 버튼이 있는 즉시 플레이 가능한 화면으로 구성합니다.
- 메인 게임 화면은 중앙의 회전하는 지구, 지구 위 이벤트 포인트, 우측 상단 업적 목록, 하단 상태/타이머 정보를 포함합니다.
- 이벤트 페이지는 자연 파괴 상황을 시각적으로 보여주고, 화면 중앙 하단에 스킬 버튼들을 제공합니다.
- 잘못된 스킬을 사용하면 이벤트 상태가 바뀌지 않아야 합니다.
- 올바른 스킬을 사용하면 해결 결과 모달을 표시하고, 실제 자연 파괴 문제와 관련된 법/뉴스/사실 정보를 보여준 뒤 업적으로 기록합니다.
- 모든 목표 이벤트를 처리하면 게임 종료 화면에서 점수, 완료 시간, 해결한 이벤트, 실패한 이벤트, 업적, 랭킹 저장 결과를 보여줍니다.

## 코드 작성 규칙

- 도메인 로직은 UI 컴포넌트 안에 과도하게 넣지 말고 `lib/` 또는 `features/` 단위로 분리합니다.
- 이벤트 종류, 스킬 매칭, 점수 계산은 데이터 주도 방식으로 관리합니다.
- Supabase 쿼리는 서버 전용 모듈로 분리하고 클라이언트 컴포넌트에서 직접 서비스 키를 사용하지 않습니다.
- 환경 변수는 `.env.local`에서 관리하고, 예시는 `.env.example`에 둡니다.
- 접근성을 위해 버튼에는 명확한 라벨을 제공하고, 주요 게임 상태는 텍스트로도 확인 가능하게 합니다.
