# GAIA 게임 디자인

## 게임 루프

1. 사용자가 이름을 입력하고 게임을 시작합니다.
2. 지구 화면에 자연 파괴 이벤트 포인트가 생성됩니다.
3. 각 이벤트는 시간이 지날수록 위험도가 상승합니다.
4. 사용자가 이벤트 포인트를 클릭하면 이벤트 해결 화면으로 이동합니다.
5. 사용자가 스킬을 선택합니다.
6. 올바른 스킬이면 이벤트가 해결되고 업적이 추가됩니다.
7. 잘못된 스킬이면 아무 일도 일어나지 않고 시간이 계속 흐릅니다.
8. 이벤트가 검정 단계까지 도달하면 실패 처리되고 사라집니다.
9. 목표 이벤트 수가 모두 해결 또는 실패되면 게임이 종료됩니다.
10. 최종 점수를 계산하고 랭킹에 저장합니다.

## 이벤트 위험도

이벤트 포인트는 아래 단계를 가집니다.

| 단계 | 색상 | 의미 |
| --- | --- | --- |
| 1 | 노랑 | 초기 발생 |
| 2 | 주황 | 확산 중 |
| 3 | 빨강 | 긴급 상황 |
| 4 | 검정 | 복구 실패, 이벤트 소멸 |

초기값은 각 이벤트마다 동일하게 둘 수 있지만, 이후 난이도에 따라 단계별 지속 시간을 다르게 설정할 수 있습니다.

## 초기 이벤트 목록

| 이벤트 ID | 상황 | 올바른 스킬 | 해결 설명 | 실패 설명 |
| --- | --- | --- | --- | --- |
| wildfire | 산불 | rain | 비를 내려 산불을 진압합니다. | 숲이 불타 생태계가 파괴됩니다. |
| ocean_trash | 바다 쓰레기 | wind | 바람으로 쓰레기를 해안 수거 지점으로 밀어냅니다. | 해양 생물이 플라스틱을 삼키거나 얽힙니다. |
| oil_spill | 기름 유출 | purify | 정화 스킬로 오염된 바다를 회복합니다. | 오염이 퍼져 해양 생태계가 손상됩니다. |
| illegal_logging | 불법 벌목 | grow | 식물을 빠르게 회복시키고 벌목지를 복원합니다. | 숲이 사라져 서식지가 줄어듭니다. |
| air_pollution | 대기 오염 | wind | 오염된 공기를 흩어내고 정화 흐름을 만듭니다. | 미세먼지와 유해가스가 확산됩니다. |
| drought | 가뭄 | rain | 비를 내려 마른 땅과 작물을 회복합니다. | 토양이 메마르고 농작물이 죽습니다. |

## 초기 스킬 목록

| 스킬 ID | 이름 | 역할 |
| --- | --- | --- |
| rain | 비 | 산불, 가뭄 같은 물이 필요한 문제 해결 |
| wind | 바람 | 바다 쓰레기 이동, 대기 오염 완화 |
| purify | 정화 | 기름 유출, 독성 오염 정화 |
| grow | 성장 | 숲과 식생 회복 |

## 스킬 판정 규칙

- 이벤트마다 `requiredSkillId`를 하나 이상 가질 수 있습니다.
- 사용자가 선택한 스킬이 `requiredSkillId`에 포함되면 해결됩니다.
- 포함되지 않으면 이벤트 상태를 변경하지 않습니다.
- 잘못된 스킬 사용에는 짧은 실패 피드백만 줄 수 있습니다.
- 점수 감점 여부는 초기 버전에서는 적용하지 않고, 난이도 확장 때 추가합니다.

## 업적 규칙

이벤트를 해결하면 업적이 추가됩니다. 업적은 지구 상태 화면과 이벤트 화면 모두에서 우측 상단부터 아래로 나열됩니다.

업적 예시:

| 업적 ID | 조건 | 표시 이름 |
| --- | --- | --- |
| first_rescue | 첫 이벤트 해결 | 첫 번째 회복 |
| fire_guardian | 산불 해결 | 숲의 수호자 |
| ocean_cleaner | 바다 쓰레기 해결 | 바다 청소자 |
| purifier | 오염 정화 이벤트 해결 | 정화의 손길 |
| perfect_run | 실패 이벤트 0개 | 완벽한 가이아 |

## 점수 계산 초안

최종 점수는 아래 요소를 합산합니다.

```txt
기본 점수 = 해결한 이벤트 수 * 100
위험도 보너스 = 노랑 해결 60 + 주황 해결 40 + 빨강 해결 20
시간 보너스 = max(0, 제한 시간 - 완료 시간 초) * 2
실패 패널티 = 실패 이벤트 수 * 50
업적 보너스 = 획득 업적 수 * 25

최종 점수 = 기본 점수 + 위험도 보너스 + 시간 보너스 + 업적 보너스 - 실패 패널티
```

점수는 0점 아래로 내려가지 않게 합니다.

## 게임 종료 조건

초기 버전에서는 `총 이벤트 수 10개`를 목표로 합니다.

- 해결 이벤트 수 + 실패 이벤트 수가 목표 이벤트 수와 같으면 게임 종료
- 이후 난이도 모드에서는 제한 시간, 이벤트 동시 발생 수, 위험도 상승 속도를 조정할 수 있습니다.
